## 1. エグゼクティブサマリー

### 1.1 製品名

**swipe-lp-creator** (AI-Powered Swipe LP Builder)

### 1.2 ミッション

> 画像がある人はアップロードだけ。ない人はAIで0から生成。最短で「売れるスワイプLP」を発行する。

### 1.3 製品概要

swipe-lp-creatorは、スワイプ型ランディングページ（LP）を簡単に作成・公開できるWebアプリケーションです。ユーザーは手持ちの画像をアップロードするか、AIによる自動生成機能を使って、数分で高コンバージョンなLPを作成できます。

## 3. ユーザーストーリー

### 3.1 コアユーザーストーリー

#### US-001: 画像アップロードによるLP作成

```
As a D2Cブランド運営者
I want to 手持ちの商品画像をアップロードしてスワイプLPを作成したい
So that 広告クリエイティブの準備時間を短縮できる
```

**受け入れ基準**:

- [ ] 複数画像の一括アップロードが可能
- [ ] ドラッグ&ドロップで順序変更が可能
- [ ] プレビューで実機表示を確認できる
- [ ] CTAボタンのカスタマイズが可能

#### US-002: AI自動生成によるLP作成

```
As a 画像素材を持たないユーザー
I want to 商品情報を入力するだけでLPを自動生成したい
So that デザインスキルがなくても魅力的なLPを作れる
```

**受け入れ基準**:

- [ ] 商品名・ターゲット・訴求ポイントの入力で生成開始
- [ ] 5〜10枚のスライドが自動生成される
- [ ] 各スライドの再生成が可能
- [ ] 生成結果の編集・差し替えが可能

#### US-003: LPの公開と計測

```
As a マーケター
I want to 作成したLPを即座に公開し、広告トラッキングタグを設置したい
So that 広告効果を正確に計測できる
```

**受け入れ基準**:

- [ ] ワンクリックで公開URLを発行
- [ ] GTM/Pixelタグの設置が可能
- [ ] Head/Bodyへのカスタムスクリプト挿入
- [ ] 公開/非公開の切り替えが可能

#### US-004: モバイル最適化された閲覧体験

```
As a LP閲覧者（エンドユーザー）
I want to スマートフォンで快適にLPを閲覧したい
So that ストレスなく商品情報を確認し、購入を検討できる
```

**受け入れ基準**:

- [ ] フルスクリーン表示（100vh x 100%）
- [ ] スムーズなスワイプ操作
- [ ] 進捗バーで現在位置を把握
- [ ] 固定CTAボタンで即座にアクション可能

### 4.2 機能詳細

#### F-001: ユーザー認証

- Firebase Authentication使用
- メール/パスワード認証
- Google OAuth認証（オプション）
- セッション管理

#### F-007: スワイプビューア（公開画面）

```
URL: /p/[lp_id]

レイアウト:
┌─────────────────────────────────┐
│  Progress Bar (Swiper)         │
├─────────────────────────────────┤
│                                 │
│                                 │
│         Slide Image             │
│         (Full Screen)           │
│                                 │
│                                 │
├─────────────────────────────────┤
│  [    CTA Button (Fixed)    ]  │
└─────────────────────────────────┘

モバイル: 100vh x 100%
PC: 中央配置 max-width 430px + ぼかし背景
```

#### F-009: AI LP生成

```
入力:
- 商品名 (必須)
- ターゲット層 (必須)
- 主要訴求ポイント (必須)
- トーン&マナー (オプション)

処理フロー:
1. GPT-5.2 → ストーリーボード生成 (5-10スライド構成)
2. 各スライド → 画像プロンプト生成
3. Gemini 3 Pro Image → 画像生成 (並列処理)
4. 結果をFirestoreに保存
5. エディタに反映

出力構成:
- Slide 1: Hook (注目を引く)
- Slide 2-3: Problem (課題提起)
- Slide 4-6: Solution (解決策)
- Slide 7-8: Benefit (ベネフィット)
- Slide 9-10: CTA (行動喚起)
```

## 5. 非機能要件

### 5.1 パフォーマンス

| 指標                           | 目標値 |
| ------------------------------ | ------ |
| First Contentful Paint (FCP)   | < 1.5s |
| Largest Contentful Paint (LCP) | < 2.5s |
| Time to Interactive (TTI)      | < 3.0s |
| Cumulative Layout Shift (CLS)  | < 0.1  |

### 5.2 可用性

- 稼働率: 99.9%
- Cloudflare Pages + Firebaseによる高可用性

### 5.3 セキュリティ

- HTTPS必須
- Firebase Security Rules
- APIキーはCloudflare Functionsで秘匿
- XSS対策（トラッキングタグのサニタイズ検討）

### 5.4 スケーラビリティ

- サーバーレスアーキテクチャで自動スケール
- Firestore: 無制限スケール
- Storage: 自動スケール

| 用途             | モデル                 | モデルID                     |
| ---------------- | ---------------------- | ---------------------------- |
| スライド画像生成 | **Gemini 3 Pro Image** | `gemini-3-pro-image-preview` |

### 6.3 モデル仕様

```yaml
gemini-3-pro-image-preview:
  input: 画像 + テキスト
  output: 画像 + テキスト
  input_tokens: 65,536
  output_tokens: 32,768
  features:
    - 高品質画像生成
    - マルチモーダル対応
    - 日本語プロンプト対応
```
